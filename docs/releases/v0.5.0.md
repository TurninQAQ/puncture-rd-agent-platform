# v0.5.0 — Local MCP Tool Servers and Algorithm Adapters

Release date: 2026-07-11 UTC

## Scope

Module 3 exposes the ten frozen algorithm contracts through three locally
runnable MCP servers and replaces a mock-only integration story with injectable
case-data, segmentation and planning/safety adapters.

This is a local Agent-engineering release.  It validates protocol, contracts,
permissions, idempotency, deadlines, trace propagation and deterministic
failure handling.  It does not claim that proprietary medical algorithms were
reimplemented or executed.

## Delivered

- MCP `2025-11-25` dependency-free JSON-RPC dispatcher;
- UTF-8 newline-delimited stdio transport;
- `tools/list` and `tools/call` for 3/3/4 logical server partitions;
- strict JSON-to-dataclass codec for all ten request contracts;
- JSON Schema 2020-12 input and URI-redacted output schemas;
- structured content plus JSON text compatibility block;
- opaque artifact handle -> trusted `ArtifactRef` resolution;
- authenticated principal/case/tool permission boundary;
- deadline and timeout mapping;
- response request/trace/version validation;
- exact ten-handler registry factory;
- optional official stable MCP Python SDK v1 adapter;
- deterministic local case-data, segmentation and planning backends;
- local ten-tool demo and stdio subprocess demo;
- detailed architecture and testing documentation.

## Adapter boundaries

### Case-data

- `CaseDataBackendPort`
- `ManifestCaseDataBackend`
- `CaseDataToolAdapter`

The local backend uses bytes/files plus explicit manifests.  A company MCS,
NIfTI, object-store and artifact-registry implementation can replace the port
without changing MCP or Agent contracts.

### Segmentation

- `SegmentationEnginePort`
- `ImageAlgorithmPort`
- `SegmentationArtifactPort`
- `SegmentationToolAdapter`
- `DeterministicSegmentationBackend`

The local backend simulates model profiles, label statistics, physical skin
thickness, engine reuse and artifact commits.  It is not a learned model or
morphology implementation.

### Planning and safety

- `PlanningKernelPort`
- `PlanningKernelManifest`
- `PlanningToolAdapters`
- `DeterministicPlanningBackend`

The local backend returns typed scripted geometry/safety/risk/traversal DTOs and
reads no voxels.  The adapter validates results and fails closed; it does not
replace company planner, EDT/collision, risk or DDA/ray kernels.

## Security evidence

- MCP input accepts only opaque artifact IDs and optional case/type assertions.
- Full internal artifact fields such as URI, checksum and metadata are rejected.
- Resolved artifacts come from a trusted resolver.
- Caller equality and case/tool allowlists are checked before handler execution.
- Model-visible results recursively project artifacts to `ArtifactPublicView`.
- URI/checksum leakage tests cover structured content, text fallback, errors and
  adapter trace records.
- Required-mask and unsafe-backend downgrade paths fail closed.
- Raw provider/native exception text is not returned across MCP.

## NOT_RUN

- installing/running the official `mcp` package;
- MCP Inspector interoperability;
- Streamable HTTP or legacy SSE transport;
- OAuth/OIDC/JWT/mTLS;
- real MCS/NIfTI parsing/writing;
- real TensorRT/CUDA inference;
- company morphology, planning, collision, risk or ray-tracing kernels;
- internal medical golden data;
- target GPU/CPU latency, throughput and memory benchmarks;
- distributed registry/idempotency/load testing.

No production accuracy, latency, throughput or GPU-memory claim is made by this
release.  Values emitted by deterministic backends are fixtures.
