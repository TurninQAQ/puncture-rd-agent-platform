# v0.7.0 — FastAPI Run Gateway and Durable Execution Recovery

Release date: 2026-07-11 UTC

## Scope

Task 07 production API runtime: secure HTTP contracts, atomic run/event store,
bounded SSE replay, and durable worker reclaim after API process failure.

## Delivered

1. Pydantic v2 request/response/error adapters with injected authority only
2. Framework-neutral atomic in-memory Run Repository (version fence, idempotency)
3. Version-scoped private event keys and fingerprints for replay/conflict
4. PostgreSQL Run/event Repository with checksum migration and COMMIT-unknown
   reconciliation
5. FastAPI 9-path Run Gateway, pre-parse Bearer/body admission, privacy views
6. Bounded SSE with strict cursors, heartbeat, token revalidation, quotas
7. PostgreSQL execution jobs + worker heartbeat/reclaim + SIGTERM recovery

## Explicit non-claims

- Production OIDC provider binding is an injection port, not a full IdP
- Cluster-wide SSE quotas and reverse-proxy slow-consumer SLAs need field proof
- Company executor side-effect atomicity / GPU cancel remain external

## Local verification baseline

- Dependency-free transport/repository unit tests always run
- FastAPI / Pydantic / PostgreSQL no-skip gates run in CI matrix jobs
