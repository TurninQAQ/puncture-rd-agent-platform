# v0.6.0 — Production Eval and Trace Scaffold

Release date: 2026-07-12 UTC

## Scope

Task 08 closes the offline evaluation and privacy-safe tracing contract for the
Agent platform. This release makes reproducible Mock reference evaluation and
span export first-class CI gates without requiring a live vendor backend.

Company medical algorithms, live GPU model services, and production OTLP
dashboards remain out of scope.

## Delivered

- OpenTelemetry-compatible facade over the existing `TraceRecorder`
- Spans for graph, node, RAG stages, model, MCP/tool, verifier, and checkpoint
- W3C `traceparent` plus MCP/HTTP/gRPC-style metadata propagation
- Strict attribute allowlist with denylist redaction
- In-memory, JSONL, composite, and optional real OTEL exporters
- Versioned Eval dataset loader (`eval-case-v1`) with JSON/JSONL support
- Deterministic RAG and Agent metrics (`metrics-v1`)
- Per-case diagnostics and baseline regression reports
- Offline CLI: `python -m puncture_agent.observability.eval_runner`
- Tracing overhead benchmark (record-only by default)
- CI: 46 zero-skip eval/tracing tests, mock eval CLI, overhead record job

## Explicit non-claims

- No live Langfuse/Phoenix/OTLP collector requirement in unit tests
- No invented quality targets for proprietary internal datasets
- No reimplementation of company MCS/TensorRT/planning algorithms
- Tracing does not provide exactly-once tool side effects

## Remaining risks

- Live exporter capacity and sampling policy must be reviewed before production
- Company OIDC principal hashing is not wired into span attributes
- Relative overhead gate on sub-millisecond mock graphs is not a production SLA
