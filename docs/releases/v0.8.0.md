# v0.8.0 — Production Eval and Privacy-Safe Tracing

Release date: 2026-07-12 UTC

## Scope

Task 08 offline evaluation and OpenTelemetry-compatible tracing scaffold.
Reproducible Mock reference evaluation and span export are first-class CI gates
without a live vendor backend.

## Delivered

- OpenTelemetry-compatible facade over `TraceRecorder`
- Spans: graph, node, RAG stages, model, MCP/tool, verifier, checkpoint
- W3C `traceparent` + HTTP/MCP/gRPC-style metadata propagation
- Strict attribute allowlist with denylist redaction
- In-memory, JSONL, composite and optional real OTEL exporters
- Versioned dataset loader (`eval-case-v1`) for JSON/JSONL
- Deterministic RAG/Agent metrics (`metrics-v1`)
- Per-case diagnostics and baseline regression reports
- Offline CLI: `python -m puncture_agent.observability.eval_runner`
- Tracing overhead benchmark (record-only by default)
- CI: 46 zero-skip eval/tracing tests, mock eval CLI, overhead record job

## Explicit non-claims

- No live Langfuse/Phoenix/OTLP requirement in unit tests
- No invented quality targets for proprietary internal datasets
- No reimplementation of company medical algorithms
- Tracing does not provide exactly-once tool side effects

## Remaining risks

- Live exporter capacity and sampling policy need review before production
- Company OIDC principal hashing is not wired into span attributes
- Relative overhead on sub-millisecond mock graphs is not a production SLA
