# v0.2.0 — Persistent Artifact Foundation

Release date: 2026-07-10 UTC

## Scope

Module 0 replaces the metadata-only Artifact Registry mock with a durable,
testable single-node implementation while preserving the public contracts.

## Implemented

- case-scoped idempotency and canonical registration fingerprints;
- SQLite persistence with WAL, foreign keys, busy timeout and explicit transactions;
- immutable lifecycle transitions and restart recovery;
- parent/child lineage persistence and persisted-cycle detection;
- secure local staging, service-side SHA-256, fsync and atomic fail-if-exists publication;
- traversal, control-character and symlink escape protection;
- orphan temporary-file cleanup;
- authorized object reads and URI-free access audit events;
- canonical JSON and deterministic tool invocation idempotency keys;
- one-process publication coordinator joining Registry and Object Store;
- CI workflow and reproducible 10k benchmark script.

## Deployment boundary

This release is safe as one application process on one trusted local storage
volume. It is not a high-availability or multi-worker storage service.

Known limitations:

- PostgreSQL and MinIO/S3 adapters are not implemented yet;
- publication locks are process-local; multi-node writers require a durable lease;
- `Principal` must be created from authenticated upstream context;
- the checked-in audit sink is in-memory and should be replaced by durable audit export;
- local files rely on host filesystem permissions and are not encrypted by this module;
- trusted reads currently materialize the complete object in memory;
- a crash after object commit but before metadata finalize is recovered by idempotent retry; a production multi-node reconciler is still required;
- the default `./var/` data directory is Git-ignored; real data should live on an externally managed private volume.

These limits are explicit so this version is not misrepresented during an interview.
