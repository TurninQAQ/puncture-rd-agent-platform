Validation date: 2026-07-15 UTC

This workstation record proves that the repository's implemented service
boundaries can execute together through real HTTP and SSE. It is not evidence
for medical algorithms, production identity, production recovery, HA, or an
SLA.

- The data/model validation Run reached `SUCCEEDED`, traversed 18 graph nodes,

- Reposting the exact data Run body returned the same Run ID and did not execute

- JSON event replay contained model, RAG, Agent graph, and terminal evidence;

- The planning/safety Run reached `SUCCEEDED`, traversed 16 graph nodes, used two

- No company document, patient data, image volume, or proprietary algorithm was
  used.

## Deliberate limitations

- The local authenticator/authorizer is an opt-in workstation implementation,
  not OIDC/IAM. The token is suitable only for loopback demonstration.
- No Artifact Access Gateway is configured, so requests use no artifact IDs and
  health remains degraded.
- Execution is synchronous. The demo does not claim API-process crash recovery;
  the production durable worker requires a recovery-safe company executor with
  its own idempotency/outbox boundary.
- OpenSearch uses deterministic development embeddings/reranking and a one-node
  yellow topology. This is not Qwen embedding quality, corpus quality, or HA
  evidence.
- Tool outputs are synthetic and must never be interpreted as clinical or
  engineering safety results.
- Reverse proxy, TLS termination for the API, production rate limits, OIDC,
  multi-process load, soak, and HTTP/PostgreSQL performance remain unrun.
